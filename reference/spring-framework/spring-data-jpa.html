Spring Data JPA

Maven Dependency

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>

Database Entity

In domain package you can define a Database Entity:

@Entity (1)
public class Author {

    @Id (2)
    @GeneratedValue(strategy = GenerationType.AUTO) (3)
    private Long id;

    private String firstName;

    private String lastName;

    @ManyToMany(mappedBy = "authors") (4)
    private Set<Book> books; (5)

    // Getters and Setters, toString(), equals(Object obj) and hashCode()

}
1 @Entity specifies that the class is a Database Entity
2 @Id specifies the primary key of an Entity
3 @GeneratedValue specifies the generation strategy for the value of primary key. AUTO indicates that the persistence provider should pick an appropriate strategy for the particular database. See üü£ GenerationType for other generation strategies.
4 @ManyToMany specifies a many-valued association with many-to-many multiplicity. The mappedBy is a field that owns the relationship.
5 ‚ö™ Set<E> is preferred here as each element will be unique 
@Entity
public class Book {

    // ...

    @ManyToMany
    @JoinTable(name = "author_book", (1)
        joinColumns = @JoinColumn(name = "book_id"), (2)
        inverseJoinColumns = @JoinColumn(name = "author_id")) (3)
    private Set<Author> authors;

    // ...

}
1 @JoinTable specifies the mapping of associations. It is applied to the owning side of an association. The name is a name of the join table
2 The joinColumns is a @JoinColumn array of the foreign key columns of the join table which reference the primary table of the entity owning the association
3 The inverseJoinColumns is a @JoinColumn array of the foreign key columns of the join table which reference the primary table of the entity that does not own the association.

Repository

When we run the application, Spring Data JPA is going to provide us a Repository implementation. It means that we only have to define a Repository interface.

public interface AuthorRepository extends CrudRepository<Author, Long> {} (1)
1 ‚ö™ CrudRepository<I,ID> is for generic CRUD operations on a Repository for a specific type

  • T is the domain type the Repository manages.

  • ID is the type of the id of the Entity the Repository manages
