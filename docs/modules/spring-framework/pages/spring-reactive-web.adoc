== Introduction

The Spring Framework is kind of *split in half*:

* *Spring Web MVC* traditionally incorporates the *Servlet API* and also *operates within Servlet Container*. It uses *blocking components*. Also, JDBC *by its nature* is also going to be blocking.
* *Spring WebFlux* can use `@Controller` / `@RequestMapping` annotations, but it also has *Router Functions* which use *functional programming paradigm*. It *does not operate* within Servlet Container.

CAUTION: Spring WebFlux is *not compatible* with Spring Web MVC.

Spring Framework 5 introduced two reactive types:

* https://projectreactor.io/docs/core/release/api/reactor/core/publisher/Mono.html[`*ðŸŸ  Mono<T>*`^] - is a https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/util/concurrent/Flow.Publisher.html[`âšª Publisher<T>`^] with *zero* or *one* element in the data stream.
* https://projectreactor.io/docs/core/release/api/reactor/core/publisher/Flux.html[`*ðŸŸ  Flux<T>*`^] - is a `âšª Publisher<T>` with *zero* or *many* elements in the data stream.

== `ðŸŸ  Mono<T>`/`ðŸŸ  Flux<T>` operations

`ðŸŸ  Mono<T>` operations:

[cols="1,3,4a"]
|===
| Modifier and Type | Method | Description

| `T`
| `block()`
| Subscribe to this `ðŸŸ  Mono<T>` and *block indefinitely* until a next signal is received. Returns that value, or `null` if the `ðŸŸ  Mono<T>` completes empty.
[,java]
----

Person person = personMono.block();
System.out.println(person);
----
[IMPORTANT]
====
Not recommended to use as it *blocks indefinitely*
====

| `Disposable`
| `subscribe(Consumer<? super T> consumer)`
| Subscribe a `âšª Consumer<T>` to this `ðŸŸ  Mono<T>` that will consume all the sequence
[,java]
----

----
[IMPORTANT]
====
This is recommended to use
====

| `<R> Mono<R>`
| `map(Function<? super T,? extends R> mapper)`
| Transform the item emitted by this `ðŸŸ  Mono<T>` by applying a synchronous function to it
[,java]
----

----

|===

`ðŸŸ  Flux<T>` operations:

[cols="1,3,4a"]
|===
| Modifier and Type | Method | Description

| `T`
| `blockFirst()`
| Subscribe to this `ðŸŸ  Flux<T>` and *block indefinitely* until the upstream signals its first value or completes.
[,java]
----

----
[IMPORTANT]
====

====

| `Disposable`
| `subscribe(Consumer<? super T> consumer)`
| Subscribe a `âšª Consumer<T>` to this `ðŸŸ  Flux<T>` that will consume all the elements in the sequence.
[,java]
----

----
[IMPORTANT]
====
This is recommended to use
====

| `<V> Flux<V>`
| `map(Function<? super T,? extends V> mapper)`
| Transform the items emitted by this `ðŸŸ  Flux<T>` by applying a synchronous function to each item.
[,java]
----

----

| `Flux<T>`
| `filter(Predicate<? super T> p)`
| Evaluate each source value against the given `âšª Predicate<T>`.
[,java]
----

    .map(Person::getFirstName)
    .subscribe(System.out::println);
----

| `Mono<List<T>>`
| `collectList()`
| Collect all elements emitted by this `ðŸŸ  Flux<T>` into a `âšª List` that is emitted by the resulting `ðŸŸ  Mono<T>` when this sequence completes, emitting the empty `âšª List` if the sequence was empty.
[,java]
----

listMono.subscribe(list -> list.forEach(System.out::println));
----

| `Mono<T>`
| `next()`
| Emit only the first item emitted by this `ðŸŸ  Flux<T>`, into a new `ðŸŸ  Mono<T>`. If called on an empty `ðŸŸ  Flux<T>`, emits an empty `ðŸŸ  Mono<T>` (*it does not signal exception!*)

| `Mono<T>`
| `single()`
| Expect and emit a single item from this `ðŸŸ  Flux<T>` source or *signal `ðŸŸ¢ NoSuchElementException` for an empty source*, or *`ðŸŸ¢ IndexOutOfBoundsException` for a source with more than one element*.
[,java]
----

    System.err.println("Error occurred in the Mono<Person> during single");
    System.err.println(throwable.toString());
}).subscribe(System.out::println, throwable -> {
    System.err.println("Error occurred in the Mono<Person> during subscribe");
    System.err.println(throwable.toString());
});
----
can output:
[,txt]
----
Error occurred in the Mono<Person> during filter
java.util.NoSuchElementException: Source was empty
Error occurred in the Mono<Person> during subscribe
java.util.NoSuchElementException: Source was empty
----

== Testing

image:https://img.shields.io/badge/Reactor%20Test%20Documentation-2088E9?logo=quickLook&logoColor[link="https://projectreactor.io/docs/core/release/reference/docs/index.html#testing",window=_blank]

Instead of:

[,java]
----
@Test
void testGetByIdNotFound() {
    Mono<Person> personMono = personRepository.getById(8);
    assertFalse(personMono.hasElement().block());
}
----

You can use

https://projectreactor.io/docs/test/release/api/reactor/test/StepVerifier.html[`âšª StepVerifier`^]::
Provides a declarative way of creating a verifiable script for an async `âšª Publisher<T>` sequence, by expressing expectations about the events that will happen upon subscription. The verification *must be triggered after the terminal expectations* (completion, error, cancellation) have been declared, by calling one of the `verify()` methods.

[,java]
----
@Test
void testGetByIdNotFoundStepVerifier() {
    Mono<Person> personMono = personRepository.getById(8);
    StepVerifier.create(personMono).expectNextCount(0).verifyComplete();
    personMono.map(Person::getFirstName).subscribe(System.out::println);
}
----
